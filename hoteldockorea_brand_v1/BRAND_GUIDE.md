**버전 1.0** · 2026년 4월

## 1. 브랜드 컨셉

### 정체성 한 문장

> 한국 3성급 이상 호텔·리조트 520여 곳의 실시간 프로모션과 주변 정보를 한 곳에서 안내하는 여행 가이드 플랫폼.

### 브랜드 서사 (로고 해설)

> **H는 머무름의 집, D는 떠오르는 달.**
> **분홍 하늘과 푸른 바다가 만나는 그곳에,**
> **한국의 호텔이 있습니다.**

로고의 의미:
- **H (채워진 보라 블록)** — 확고한 집, 머물 곳, Hotel
- **D (흰 반달 외곽)** — 떠오르는 여행의 순간, 여행의 기록, Doc(ument)
- **배경 사인커브** — 하늘과 바다, 한국 자연의 풍경
- **파스텔 톤** — 첫 새벽의 여명, 여행의 시작

## 2. 색상 팔레트

### 주요 색상

| 이름 | HEX | RGB | 용도 |
|---|---|---|---|
| Primary Purple | `#534AB7` | `83, 74, 183` | 로고·프레임·H 채움·브랜드 텍스트 |
| Light Purple | `#7F77DD` | `127, 119, 221` | 보조 텍스트·서브타이틀 |
| Pastel Pink | `#FADDDC` | `250, 221, 220` | 배경 상단 (하늘) |
| Pastel Blue | `#D4E4F2` | `212, 228, 242` | 배경 하단 (바다) |

### D 내부 투명도

```css
D 외곽선: #534AB7, 굵기 1px
D 내부 채움: #FFFFFF, fill-opacity: 0.5
```

### 색상 사용 원칙

- Primary Purple(#534AB7)은 H·D 외곽선·프레임 테두리 **세 요소에만** 사용
- 로고 외 본문에서 보라를 쓸 때는 Light Purple(#7F77DD) 권장
- 흰 배경 페이지에서는 **컬러 버전**, 어두운 배경에서는 **흰색 단색 버전** 사용

## 3. 최소 크기 규칙

| 용도 | 최소 크기 | 권장 크기 |
|---|---|---|
| 웹 헤더 | 40×40px | 56×56px |
| 파비콘 | 16×16px | 32×32px |
| 명함 | 15×15mm | 20×20mm |
| 인쇄물 | 15mm | 25mm 이상 |

**16px 미만으로는 사용하지 않습니다** — 반달 D의 곡선이 깨져 인식 불가능해집니다. 더 작은 크기가 필요하면 **H만 단독**으로 사용하세요(보라 H 블록을 아이콘처럼).

---

## 4. 여백(Safe Area)

로고 주변에는 **로고 너비의 10% 이상** 빈 공간을 확보해야 합니다.

```
로고 크기 100px → 여백 10px 이상
로고 크기 40px → 여백 4px 이상
```

여백 안에 다른 텍스트·이미지·배경 요소가 침범해서는 안 됩니다.

## 5. 사용 금지 예시

### ❌ 다음과 같이 사용하지 마세요

1. **색상 임의 변경** — "앰버 D" 같은 변형 불가
2. **비율 왜곡** — 정사각형 프레임을 직사각형으로 늘리기 불가
3. **회전** — 로고는 항상 수직으로 사용
4. **그림자·광택 추가** — 플랫 디자인 원칙 유지
5. **H·D 간격 임의 조정** — 내부 비율은 고정
6. **투명도 변경** — D의 fill-opacity 0.5는 고정값
7. **배경과 유사한 색 위에 사용** — 대비가 부족한 배경 피하기

### ✅ 권장 사용 방법

- **흰 배경 / 연한 회색 배경** → 컬러 버전
- **진한 보라·검정 배경** → 흰색 단색 버전
- **흑백 인쇄** → 검정 단색 버전
- **1색 엠보싱·워터마크** → 보라 단색 버전

### 브랜드 텍스트

- **로고 바로 옆 "HotelDocKorea"** → Sans-serif 굵기 600, 색상 `#534AB7`
- **서브타이틀** → Sans-serif 굵기 400, 색상 `#7F77DD`
- 웹 권장 폰트: `system-ui, -apple-system, "Segoe UI", sans-serif`

- 제목 H1: 32px, 굵기 700
- 제목 H2: 24px, 굵기 600
